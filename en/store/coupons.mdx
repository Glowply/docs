**Coupons (القسائم)** are discount codes you create to run promotions and reward shoppers. A customer types the code into your **cart** or **checkout**, and your Store applies the discount to their order. You decide what each code does — a percentage off, a fixed amount off, when it expires, and who can use it.

You'll find coupons in your **site dashboard** under **Store (المتجر) → Coupons (القسائم)**. This is different from your **account portal**, which is only for billing.

<Info>
  Not sure how to open your site dashboard? See [Site dashboard overview](/en/site/dashboard-overview). Your coupons live in the **site dashboard**, not the account portal.
</Info>

<Note>
  A **coupon** is a code customers enter at checkout. If you just want to lower a single product's price with no code, set a **sale price** on the product instead — see [Products](/en/store/products).
</Note>

## The Coupons list

Open **Store → Coupons** to see every code you've created in one list. Each row shows the code, the discount type and amount, how many times it's been used, its usage limit, and its expiry date. From here you can:

- **Add** a new coupon.
- **Open** a coupon to edit it.
- **Search** for a code by name.
- **Delete** a code you no longer want to offer.

<Tip>
  Pick codes your customers can read and type easily, like `WELCOME10` or `EID25`. Short, memorable codes work best in posts, messages, and ads.
</Tip>

## Discount types

Before you create a code, decide how the discount should apply. This choice controls what the code takes off the order.

<CardGroup cols={2}>
  <Card title="Percentage discount" icon="percent">
    Takes a **percentage** off the cart — for example **10%** off everything. The saving grows with the order size.
  </Card>
  <Card title="Fixed cart discount" icon="cart-shopping">
    Takes a **fixed amount** off the whole cart — for example **50 EGP** off the order total, regardless of how many items it holds.
  </Card>
  <Card title="Fixed product discount" icon="tag">
    Takes a **fixed amount** off **each eligible product** in the cart — for example **20 EGP** off every qualifying item.
  </Card>
</CardGroup>

<Note>
  All amounts are in Egyptian pounds (EGP, ج.م). A **percentage** discount uses a number like `10` for 10%; a **fixed** discount uses an amount like `50` for 50 EGP.
</Note>

## Create a discount code

Follow these steps to create a coupon and make it active. You can come back and edit any of these settings later.

<Steps>
  <Step title="Open Coupons and add a new one">
    In your site dashboard, go to **Store → Coupons**, then choose to **add a new coupon**. A blank coupon opens.
  </Step>
  <Step title="Enter the code">
    Type the **coupon code** customers will use, such as `WELCOME10`. This is exactly what they enter at checkout, so keep it short and easy to read.

    <Tip>Codes usually aren't case-sensitive, but it's clearest to share them in one consistent style — for example all capitals.</Tip>
  </Step>
  <Step title="Choose the discount type">
    Pick **percentage**, **fixed cart**, or **fixed product** — see [Discount types](#discount-types) above for what each one does.
  </Step>
  <Step title="Set the amount">
    Enter the discount **amount**. For a percentage, enter the percent (such as `10`). For a fixed discount, enter the amount in EGP (such as `50`).
  </Step>
  <Step title="Add a description (optional)">
    Write a short **description** to remind yourself what the code is for — for example *Ramadan launch, 10% off everything*. Customers don't see this; it's just for you.
  </Step>
  <Step title="Set an expiry date (optional)">
    Choose an **expiry date** so the code stops working on its own. After that date, the code is no longer accepted at checkout.

    <Tip>Always set an expiry on a limited-time promotion so you don't have to remember to turn it off.</Tip>
  </Step>
  <Step title="Add restrictions and limits (optional)">
    Set a **minimum spend**, restrict the code to certain **products or categories**, or cap how many times it can be used. See [Restrictions](#restrictions) and [Usage limits](#usage-limits) below.
  </Step>
  <Step title="Save and publish">
    Save the coupon to make it active. Test it by entering the code at your own checkout to confirm the discount applies as you expect.
  </Step>
</Steps>

<Check>
  Add an eligible item to your cart, enter the code at checkout, and confirm the discount, the expiry, and any minimum spend all behave the way you intended.
</Check>

## Restrictions

Restrictions decide **when** a code is allowed and **what** it applies to. Use them to protect your margins and keep a promotion focused.

| Restriction | What it does |
|---|---|
| **Minimum spend** | The cart total must reach this amount before the code works — for example *spend 300 EGP to use it*. |
| **Maximum spend** | The code only works up to a cart total — useful to cap a generous discount. |
| **Products** | Limit the discount to specific **products**, so it only applies when those items are in the cart. |
| **Exclude products** | Stop the code from applying to chosen products — for example exclude items already on sale. |
| **Categories** | Limit the discount to whole **categories**, such as *Dresses* — handy for a category-wide promotion. |
| **Exclude categories** | Stop the code from applying to chosen categories. |
| **Exclude sale items** | Prevent the code from stacking on products that already have a **sale price**. |

<Warning>
  By default a discount can combine with a product's existing **sale price**. If you don't want a code to stack on items already discounted, turn on **exclude sale items** so shoppers can't get both at once.
</Warning>

<Tip>
  To run a sale on one part of your shop — say 20% off all dresses — set a **percentage** discount and limit it to the **Dresses** category. Organize products into categories first so the restriction has something to target. See [Categories & tags](/en/store/categories-tags).
</Tip>

## Usage limits

Usage limits cap **how often** a code can be redeemed, so a promotion can't be overused.

| Limit | What it controls |
|---|---|
| **Usage limit per coupon** | The total number of times the code can be used across all customers — for example the first **100** orders only. |
| **Usage limit per customer** | How many times one customer can use the code — set it to **1** for a one-per-person offer. |
| **Limit to items in the cart** | Caps how many eligible items in a single cart the discount applies to. |

<Note>
  Once a coupon reaches its **usage limit**, it stops working automatically — even if its expiry date hasn't passed yet.
</Note>

## Edit, disable, and delete coupons

You can change a code at any time, and your Store applies the update right away.

- **Edit a coupon** — open it from the Coupons list, change the amount, restrictions, or expiry, and save.
- **End a promotion early** — set the **expiry date** to today (or in the past) so the code stops being accepted.
- **Delete a coupon** — remove a code you no longer offer. Past orders that already used it aren't affected.

<Warning>
  **Deleting** a coupon is permanent. If you might run the same promotion again, leave the coupon in place and just let it **expire** instead — you can reactivate it later by extending the date.
</Warning>

## How customers use a coupon

Your customers don't see any of the dashboard. On your live site, they add items to the **cart**, then enter your code in the **coupon** or **discount code** field in the cart or at **checkout**. When the code is valid, the discount is applied to the order total before they pay.

<Note>
  If a code is rejected, it's usually because it has expired, reached its usage limit, the cart hasn't met the **minimum spend**, or the cart doesn't contain the **products or categories** the code is limited to.
</Note>

## Common questions

<AccordionGroup>
  <Accordion title="What's the difference between a coupon and a sale price?">
    A **coupon** is a code the customer types at checkout to get a discount. A **sale price** lowers a product's price for everyone, with no code needed. Use a coupon for promotions you share with specific people or campaigns; use a sale price for an open discount on one item. See [Products](/en/store/products).
  </Accordion>
  <Accordion title="Can a coupon apply to only one category?">
    Yes. Limit the coupon to one or more **categories** under its restrictions, and it will only discount items from those categories. Set up your categories first so there's something to target. See [Categories & tags](/en/store/categories-tags).
  </Accordion>
  <Accordion title="How do I make a code usable only once per person?">
    Set the **usage limit per customer** to **1**. Each customer can then redeem the code a single time. To cap the promotion overall, also set a total **usage limit per coupon**.
  </Accordion>
  <Accordion title="Can customers use more than one coupon on the same order?">
    It depends on how each coupon is set up. To keep things predictable, plan your promotions so codes don't unexpectedly stack, and use **exclude sale items** to stop a code from combining with products already on sale.
  </Accordion>
  <Accordion title="My coupon isn't working — what should I check?">
    Confirm the code hasn't **expired** or hit its **usage limit**, that the cart meets any **minimum spend**, and that the cart contains the **products or categories** the code is limited to. Test the exact code at your own checkout to see what happens. Still stuck? See [Troubleshooting](/en/troubleshooting).
  </Accordion>
</AccordionGroup>
